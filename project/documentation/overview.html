Project Overview
This page is dedicated to an overview of the project "World of Wonders", a website, developed individually as part of the coursework for the CS5610 - Web Development course, under Professor Richard Rasala. This course is part of my curriculum for the MS in Computer Science degree at Northeastern University, Boston.
Objective
The objective behind this website is to enable users to appreciate the cultural heritage of our common humanity and explore such places and structures present around the world.
On a personal level, I have a passion for photography and exotic places, including those that have a rich history and significance. The love of beautiful photographs and desire to visit amazing places led me to think about the World Heritage sites, and develop a web site based on them.
Functionalities
The main functionalities of this website can be summarized as follows -
  • Ability to browse through some of the most popular World Heritage sites on Google Maps
  • Retrieve basic information about a WHS when a user clicks on a marker representing its location
  • To be able to view extended information about a WHS by accessing its Wikipedia link given on the site
  • Enjoy gorgeous photographs of a WHS by professionals and amatuers alike
Providing such functionalities listed above has been possible through JavaScript implementation of the following APIs -
  • Google API for Google Maps
  • Flickr API for photographs on Google Maps infoWindows
  • 500px API for photographs of WHS
  • MediaWiki API for retreival of basic information about a WHS from Wikipedia
A thorough technical explanation of implementation of these APIs is given here in the project documentation.
Software platform
This website is purely based on HTML/CSS. All client side logic has been implemented using jQuery. This website does not require the use of server side code, nor does it call any web services. Therefore, I have avoided using .NET specific technologies provided by ASP.NET.

The data presented on this web site comes from two sources - APIs and a 'flat-file'. Photos & information on WHS sites come from respective APIs. Their location information required by Google Maps to display them comes from a text file (flat-file). Specific tag-information, necessary to fetch relevant information from Flickr, 500px and Wikipedia, are also stored in this text-file.

During my discussion with Prof. Rasala about the project, I was recommended to use a text file for storing such data instead of using a database. The reasons for that are as follows -
  • Such data is not relational by nature, so no primary key or foreign key is required
  • Data volume is not humongous, and therefore can be easily managed
  • Saves unnecessary overhead in accessing the database on server, and maintaing a connection with it, etc
  • Avoids use of server access when it is quite unnecessary
  • Very easy to add/remove records
  • It is easy to parse data on text files using the built-in AJAX capability of jQuery
A detailed description of how data is arranged in the text file and how I access it using jQuery is given here in the project documentation.
Browser compatibility

This website works well across latest versions of all major browsers - IE9, Mozilla Firefox & Google Chrome.

Made for iPhone
This website runs well on computer screens. Apart from that, I have ensured a beautiful experience on the iPhone. I have used CSS media queries to change the layout and styles of elements so that the iPhone user is focused on the content, and does not have to worry about usability and size of elements.
