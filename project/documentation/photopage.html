The Photo page

This page shows the photos of a particular World Heritage site that was selected by clicking on the Google Map marker on the Mapview page. The photos are large sized and are shown in a slide show with previous and next buttons. There are two jQuery files associated with this page: slideshow.js and photos.js. Photo titles

This page features a slideshow that displays 10 photos. Each photo has a title and the display-name of the photographer. The slideshow has previous and next buttons that cycle through each of them. Once a user goes to the next or previous photo, the title and display-name of the photographer are displayed on top.

When the user clicked on the View More Photos button, jQuery passed on the photo tag information to this page in a query string. The jQuery code for this page extracts this information and calls the 500px API fetch the ten photos. It also extracts data from the WHS_list text file and gathers all the information into a two dimensional array. It uses this to match the tag passed into the query string with the tags present in the two dimensional array to verify the validity.

Once the 500px API is called and the data is fetched, all the photo URLs are put into an array. At the same time, the title and display-name of the photographer is also collected from the JSON data and put into this array. Each photo URL in the array is then inserted into the img tags that make up the slideshow, and their titles and display-names are inserted into corresponding div tags on top.

In case there is an error in retrieving data from the API, jQuery fires off an error function that alerts an error message along with the error code.

Slideshow

The HTML for the slideshow consists of a div container, and figure tags for each of the ten photos. Each figure tag contains within it an imgtag. Initially, the src attributes of these image tags are empty. The containing div tag is relatively positioned, while each figure tag is absolutely positioned. Initially the first figure tag is given a class show that displays it. The same thing for the div tags for the title and display-name. Once the user clicks on either the previous or next button, jQuery changes to the previous or next photo respectively, removes the show class from all figure tags, and reapplies it to the current photo. The same treatment is given to the title div and the display-name div.

Design for iPhone

The layout for the iPhone is quite different from that of desktop browsers. Instead of having a photo slideshow, I chose to have a simple layout of vertically stacked images one after the other, in a single column. This design choice ensures that the user can look at the photos in their maximum size fitting the iPhone screen. A slideshow would have caused the photo size to be smaller to make room for the previous and next buttons. Also, the simple layout in a single column is aesthetically better suited to small screens.

The photo title and display-names of the photographers are inserted in the same manner as in the case of the desktop version.
