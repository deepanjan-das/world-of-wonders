World of Wonders - Homepage
The home page is the one that users will see first when they visit the web site.
Features
This page features a fading slideshow of photos of four World Heritage sites. There is a highlighted feature suggesting that the user can view photos of WHS and explore their location on Google Maps. Below that, the user is presented with a "Start Here" button, that launches them into the exprience of exploring WHS locations featured on this site.

There is a top navigation area at the top-right corner (for desktop browsers) and just below the site logo (for iPhone browsers).

The layout of the site features a top header for logo and navigation (another one for navigation on iPhone), a fading slideshow container, a two column layout for advertising the photo and Google Maps experience, and a single column layout for "Start Here" button and the footer.
Implementation of fading slideshow
The slideshow automatically fades in-between photos. This is made possible using CSS and jQuery. In CSS, I put the img tags inside a div container. The position of the containing div is relative and those of the img tags are absolute so that the images are stacked one on top of the other. The jQoery code cycles through the photos using the setInterval() built-in function. It uses a fadeOut and fadeIn values of 1000ms. The jQuery function also hides all the images at first, by the following code: $("#autoSlide > div:gt(0)").hide(); and then shows the current one it cycles through. The time interval between each subsequent image is set as 4000ms. The jQuery code can be found here, and the CSS can be found here.
Design for iPhone
The home page is redesigned for iPhone using CSS @media queries. I have covered the portrait as well as the landscape orientation modes by using the following @media queries -
  • Portrait mode: @media screen and (max-width: 320px)
  • Landscape mode: @media only screen and (min-width: 480px) and (max-width: 1024px)
The media queries change the two-column layout of the mid-section that advertise the photo and Google Maps features, into a single column layout. Also, for the iPhone, the usual top navigation menu at the top-right disappears, and in its place appears a new navigation bar just below the logo. The logo itself is enlarged. For this, I toggle the display: none and display: block display modes through CSS. By default, in a computer monitor, the mobile navigation bar is hidden. On an iPhone, the media query detects the device, and hides the regular navigation bar while revealing the mobile navigation bar, and makes further changes to layouts and element sizes.

The CSS for the same can be found here. The classes concerning navigation bar are -
  • Regular desktop navigation bar: topmenu, topmenu-li
  • iPhone navigation bar: mobile-nav, mobile-nav-display-toggle, mobile-top-nav, mobile-top-nav-li
